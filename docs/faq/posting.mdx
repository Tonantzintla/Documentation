## So how do I ask my question such that it's easy to help me?

To maximise people's ability to help you with your question, here are some tips for formatting a technical question.

    <details>
      <summary>Do not just say...</summary>
      <ul>
        <li>
          "Hello"
          <ul>
            <li>
              You're trying to be polite by not jumping right into the request, like you would do in person or on the phone. Chat is neither of those things, and typing is much slower than talking. Instead of being polite, you are just making the other person wait for you to phrase your question. See <a href="https://nohello.com">nohello</a> for more information.
            </li>
          </ul>
        </li>
        <br />
        <li>
          "My code doesn't work"/"[thing] didn't fix it"
          <ul>
            <li>This description is far too vague to do anything about, and requires long litanies of follow-up questions. People may choose to merely ignore your question because you did not even ask one.</li>
          </ul>
        </li>
        <br />
        <li>
          "Can someone help me?"/"Can I ask a question?"
          <ul>
            <li>This is in the same vein as the above two examples. This is a non-question that people may choose to just ignore, as finding out the actual question requires a lot of added time and effort.</li>
          </ul>
        </li>
        <br />
        <li>
          "Any [subject] experts here?"
          <ul>
            <li>Asking if there are any "experts" to help you implies very negative things. You're walling people who do not self-identify as experts (which is most people) off from answering your question, even though the question may not need any specific expertise whatsoever to answer. In general, phrasing your question in this way just discourages people from ever actually answering it.</li>
          </ul>
        </li>
        <br />
        <li>
          "Please just give me the code to do [thing]"
          <ul>
            <li>"Give someone a fish, and you'll feed them for a day. Teach someone to fish, and you'll feed them for a lifetime." Spoonfeeding you answers is in nobody's best interest. If you are given the answer on a silver platter, you'll be back the next day with the same problem without even realizing it's the same problem, because you learned nothing. Helping you understand the logic will help you understand how to arrive at the answer yourself, and will help you be able to write solutions to the same problem yourself in the future.</li>
          </ul>
        </li>
      </ul>
    </details>

      <summary>When posting your question, do not...</summary>
      <ul>
        <li>
          Crosspost
          <ul>
            <li>Please refrain from posting your question in multiple channels, or linking the channel elsewhere. If someone knew e.g. Python, they would be in the Python channel; asking in irrelevant channels is just spam. Be patient in waiting for an answer, as it may take time for someone with adequate knowledge to answer your question. A well-structured question as per directives above will help you get help quicker.</li>
          </ul>
        </li>
        <br />

            <li>

If what you are doing is so sensitive that you cannot even disclose basic code (which is often very generic and non-identifying), you would probably not be asking your question on the internet at all.
            </li>
          </ul>
        </li>

      </ul>

### How can I post code on Discord?

Discord has a character limit of 2000 characters per message. If your code is longer than that, you can use a code-sharing service, like GitHub or Hastebin. You will no longer be automatically muted for posting long blocks of code, but please still choose one of the options above so your post isn't flooded with long blocks of code and will make it easier for users to actually find the question you are asking.

#### Code blocks

Discord supports code blocks, which are a way of formatting code in a way that is easy to read.

<Tabs>
  <TabItem value="inlineblocks" label="Inline code blocks" default>
    <p>First off, you have inline blocks that can be created using backticks (`) on each side of your code.</p>
    <p>
      For example, `print("Hello, world!")` will produce <code>print("Hello, world!")</code>.
    </p>
    <p>These blocks cannot have syntax highlighting and will look odd with multi-line content, so it's probably wise to only use them for short, one-line snippets.</p>
  </TabItem>
  <TabItem value="multilineblocks" label="Multiline code blocks">
    <p>Then you have the standard, multiline blocks that also include syntax highlighting. You can make use of those by surrounding your code with three backticks (`) on each side. You can also enable syntax highlighting by typing the name of the language right after the first three backticks.</p>
    <p>For example,</p>
    <CodeBlock language="lua">
{`\`\`\`lua
print("Hello, world!")
\`\`\``}
    </CodeBlock>
    <p>will produce</p>
    <CodeBlock>print("Hello, world!")</CodeBlock>

  </TabItem>
